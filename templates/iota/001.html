Model-view-controller

Model-view-controller (MVC) is a software design pattern commonly used for developing user interfaces that divides the related program logic into three interconnected elements. This is done to separate internal representations of information from the ways information is presented to and accepted from the user.

The state of the system is represented by the model parameters.

State implies state of the system. Props refer to the configuration state of the view components. Properties are generated mapping over model parameters and reducing the model to a set of properties at time \(t\). The map from \(\textbf{M}_t \xrightarrow[Reducer]{\text{View}} \textbf{P}_t \). is performed whenever the model changes.

A controller responds to events and modifies the model.

A controller is aware of the system's model and changes the model according to business logic which depends on the current state and the action being requested by the event. A state transition function, \(\textbf{A}()\) maps the system state from some time in the past \((t-\Delta T) \) to current \(t\) where \(\textbf{A}_t\) is the action on the model at time \(t\): $$ \textbf{M}_{t} = \textbf{A}_t(\textbf{M}_{t-\Delta T}) $$

The following concepts must be considered as a whole system to provide a consistent user experiecnce:

  • State
  • Events
  • Controller
  • Model
  • Props
  • View
  • Time

View components subscribe to model changes and recieve properties.

A controller never updates the view. Instead, view components register with the model to subscribe to updates. The model generates view updates whenever data changes. In this way, the view is automatically updated as a result of the controller modifying the data.
